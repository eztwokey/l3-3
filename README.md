# CommentTree — древовидные комментарии с навигацией и поиском

## Описание

Сервис для работы с древовидными комментариями, поддерживающий неограниченную вложенность, полнотекстовый поиск, постраничную навигацию и каскадное удаление.

## Технологии

- **Go 1.25** — язык разработки
- **Gin** — HTTP-фреймворк
- **PostgreSQL 16** — хранилище комментариев, рекурсивные CTE, полнотекстовый поиск (tsvector + GIN)
- **wbf** — внутренняя библиотека (dbpg, logger, config)
- **Docker Compose** — оркестрация инфраструктуры

## Архитектура
```
HTTP-запрос → API (хендлеры) → Logic (бизнес-логика) → Storage (PostgreSQL)
```

- **API** — принимает HTTP-запросы, парсит параметры, возвращает JSON
- **Logic** — валидация, сборка дерева из плоского списка, координация storage-методов
- **Storage** — SQL-запросы: рекурсивные CTE, полнотекстовый поиск, пагинация
- **Interfaces** — контракты для storage

## Схема БД

**comments** — одна таблица, дерево через self-reference:
| Поле | Тип | Описание |
|------|-----|----------|
| id | SERIAL | первичный ключ |
| parent_id | INT (FK → comments.id) | родитель (NULL = корневой) |
| author | VARCHAR(100) | автор |
| text | TEXT | текст комментария |
| created_at | TIMESTAMP | дата создания |
| search_vector | TSVECTOR (GENERATED) | автоматический индекс для полнотекстового поиска |

`ON DELETE CASCADE` — удаление родителя каскадно удаляет всех потомков.

## API

| Метод | Путь | Описание |
|-------|------|----------|
| POST | /comments | Создать комментарий |
| GET | /comments | Список с пагинацией, сортировкой, поиском |
| GET | /comments/:id | Комментарий со всеми потомками (дерево) |
| DELETE | /comments/:id | Удалить комментарий и всех потомков |

### GET /comments — query параметры

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| parent_id | Фильтр по родителю | NULL (корневые) |
| page | Страница | 1 |
| page_size | Размер страницы | 20 |
| sort | Сортировка: created_at_asc, created_at_desc | created_at_desc |
| search | Полнотекстовый поиск | — |

### Полнотекстовый поиск

Используется PostgreSQL `tsvector` с русской морфологией. Поле `search_vector` генерируется автоматически при вставке. Индекс GIN ускоряет поиск. Запрос «комментарий» найдёт «комментарии», «комментариев» и т.д.

### Рекурсивное дерево

GET /comments/:id использует `WITH RECURSIVE` CTE для получения всех потомков на любую глубину одним SQL-запросом. Logic собирает плоский список в дерево за O(n).

## Запуск
```bash
# 1. Поднять PostgreSQL
make start

# 2. Скопировать конфиг
cp config/config.yaml config/config.yaml

# 3. Запустить (миграции выполнятся автоматически)
make run
```
